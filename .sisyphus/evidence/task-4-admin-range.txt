Task 4: Admin explicit date range still works independently
=====================================================================

Evidence: Test "uses explicit startDate/endDate params that differ from both the frozen UTC day and EVENT_DATE"

Setup:
- Frozen UTC day: "2026-01-20" (via withFrozenTime)
- EVENT_DATE env: "2026-01-15" (configured in createEnv)
- Explicit query params: startDate=2026-01-14, endDate=2026-01-16

Records seeded:
- scan-explicit-in-range: event_date=2026-01-15 (within explicit range)
- scan-explicit-on-frozen-day: event_date=2026-01-20 (frozen UTC day, outside explicit range)
- scan-explicit-before-range: event_date=2026-01-13 (before range)

Assertions:
- /records?startDate=2026-01-14&endDate=2026-01-16 returns ONLY the in-range record
- dateFilter reflects the explicit params, NOT the frozen UTC day
- /export.csv?startDate=2026-01-14&endDate=2026-01-16 returns ONLY the in-range CSV row
- The frozen-UTC-day record is excluded despite time being frozen to that day
- The EVENT_DATE record IS included because it falls within the explicit range
