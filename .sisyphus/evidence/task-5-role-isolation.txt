Task 5: Role Isolation Verification
====================================

Purpose: Verify that secret tokens are properly scoped to their role and cross-role misuse fails.


Test Case 1: Student route with mentor token
===========================================
Command:
  GET https://absen-qr.rif42.workers.dev/student/mentor-mohammad-ariq-nazar-s-si-m-biomed/api/me

Context:
  - Route: /student/... (student route)
  - Token: mentor-mohammad-ariq-nazar-s-si-m-biomed (belongs to mentor)
  - Expected: Should be rejected as the token belongs to a different role

Expected:
  - 404 Not Found status
  - JSON response: { "error": "Not found" }

Actual Response:
  HTTP Status: 404 Not Found
  {
    "error": "Not found"
  }

Verdict: PASS
  - Student route correctly rejects mentor token
  - Status code is 404 as expected
  - Error message matches expected format
